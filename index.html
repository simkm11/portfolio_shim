프로젝트 소개


뚜벅뚜벅 : 여행 전문 커뮤니티

"웹사이트 풀스택 과정" 진행중 프로젝트로 여행 전문 커뮤니티인 "뚜벅뚜벅"을 구현하는 프로젝트를 수행을 했습니다.
뚜벅뚜벅은 국내에 여행을 주제로 전문성을 띄고있는 커뮤니티를 찾을 수가 없었고, 여행을 즐기는 사람들에게 보다 전문적인 여행 정보를 제공하고자 기획 한 사이트 입니다.
프로젝트는 프론트엔드, 백엔드 총 2번의 발표로 진행을 하였고, 과정을 수행하는 학생과 수업하시는 선생님의 심사를 통해 프론트엔드, 백엔드 두 번 모두 9개팀 중에 1위인 최우수상에 선정 되었습니다.

프로젝트 테이블 제작

팀원 들과 회의를 통하여 테이블을 제작 하였습니다.
각각 글쓰기 테이블, 공공기관게시판 테이블, 회원정보 테이블, 북마크 테이블로 나눴으며
각자의 페이지에 필요한 데이터를 가져오기 위하여 기본키를 설정하고 각 테이블에 외래키를 주어
데이터를 가져오도록 제작하였습니다.

모달창을 이용한 로그인 폼 구현

이용자에게 보다 심심하지않는 로그인 페이지를 구현하고 싶었고
밋밋한 로그인 페이지가 어떻게 재미있게 표현이 될까라는 고민 끝에 모달창을
이용해 로그인 폼을 구현하기로 결정하고 구현을 하였습니다.

로그인폼 코드와
     로그인 일치 여부 코드

로그인 폼 HTML 코드와 Jquery를 이용한 모달창 fadein ,fadeout 구현해 보았습니다.
또한 Jquery 와 Ajax를 사용하여 로그인 정보 일치여부 구현을 하였습니다.
로그인 폼 HTML 구현을 하면서 박스가 자꾸 밖으로 나가 애를 먹었지만 테이블태그를 사용하여
레이아웃을 잡았고,Jquery와 Ajax 작동원리에 대하여 이해를 하고 구현을 하였습니다.

글쓰기 페이지

글쓰기 페이지는 부트스트랩을 통하여 에디터를 불러와 구현을 하였고.
대표이미지로 사용하기 위한 이미지 파일을 등록 할 수 있도록 파일 타입을 주었습니다.

Controller 와 Service

대표 이미지 파일 저장을 위하여 UUID를 사용하여 파일 이름 중복을 방지 하였습니다.
Controller 에서 Service 를 호출을 합니다.
호출 받은 Service는 DAO를 호출 합니다.

DAO 와 Mapper

Service에서 호출 받은 DAO는 Mapper의 SQL문을 실행시켜 해당되는 값을 가지고
다시 Service로 이동 Service는 다시 Controller로 이동 전달 받은 Controller는 해당되는 값을 db에 저장하고,
모든 작업을 마친 컨트롤러는 메인으로 이동 시켜줍니다.









진행중인 프로젝트


시선 : 온라인 전시회 사이트

온라인 전시회 사이트 "시선" 프로젝트
온라인 전시회 "시선"은 그림을 좋아하고 사진을 좋아하는 고객에게 작가가 될 수 있는 기회를 제공 하는 사이트 입니다.
온라인 개인 전시회나 콜라보레이션 같은 단체 전시회를 열도록 공간을 대여해주며 개인 작품을 판매도 할 수 있도록 제공을 해줍니다.
사이트의 효과로는 예술에 대한 진입장벽을 높이며 온라인 전시회 관객들에게는 더욱 다양한 작가들 작품을 감상 할 수 있습니다.
현재 진행중인 프로젝트이고 아직 마무리를 하지 못하였습니다.
데이터를 가져오는 백엔드 작업은 대부분 끝난 상태이며 프론트엔드 와 목록에 검색 기능과 필터 기능은 구현 예정에 있습니다.

목록 forEach 사용
     데이터베이스에서 데이터 출력

forEach 를 사용하여 카드목록을 반복하게 만들고 각자의 카드에 Datebase의 데이터를 넣어
출력을 하게 하였습니다.

Contoller

UUID를 사용한 파일 이름 중복 방지 및 파일 이름 전달

UUID를 사용하여 파일 이름 중복을 방지하였고
전 프로젝트보다 나아가 변경 된 파일 이름을 데이터베이스에 저장을 시켜
등록과 동시에 목록페이지에서 바로 이미지 사진과 데이터를 출력 할 수 있게
설계하였습니다.


     Service,DAO,mapper

Controller에서 호출 받은 Service는 DAO를 호출 합니다.
호출 받은 DAO는 Mapper에서 SQL문을 실행시켜 해당 값을 가지고
다시 역순으로 Controller로 향해 값을 데이터베이스에 저장 시킵니다.

전 프로젝트에서는 데이터베이스에 저장 (등록페이지) 구현을 했다면,
이번 프로젝트에서는 데이터베이스에 저장(등록페이지) 및 데이터베이스에서 값을 출력 시키는
(목록페이지)를 구현을 하였습니다.

앞으로 목록페이지에서 검색 기능을 통하여 검색한 값에 해당 데이터만 불러오는 필터 기능을 구현 할 예정이고, 메인페이지에서는 최근 게시글을 보여주는 기능을 구현 할 예정입니다.
